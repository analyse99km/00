WHAT ZENO DOES IN ONE RUN
1. GitHub Actions starts the workflow in analyse99km/v1.
2. The workflow checks out the repo and hydrates the Chromium profile from analyse99km/cpr.
3. The workflow builds and runs the Docker container on Ubuntu 22.04.
4. zeno_v1.py loads zeno_config.txt.
5. Config values are converted into environment variables:
TOPIC becomes ZENO_MISSION_TOPIC.
TARGET_REPO becomes ZENO_TARGET_REPO.
Other keys become ZENO_* variables.
6. Zeno starts the runtime engine, memory system, browser controller, trend hunter, and GitHub operations layer.
7. Because DEEP_AUDIT_ENABLED=1, Zeno enters deep account audit mode before normal loop work.
8. Zeno creates mission-based X/search queries from the configured topic only.
9. Zeno searches for public high-signal posts related to the mission topic.
10. Zeno extracts candidate account handles from those posts.
11. Zeno skips accounts already audited in memory.
12. Zeno continues collecting candidates until it reaches the configured minimum quota.
13. For each selected account, Zeno collects a profile summary.
14. For each selected account, Zeno collects up to the configured number of public posts.
15. For each collected post, Zeno collects up to the configured number of public comments/replies.
16. Zeno saves account, post, and comment records locally under data/deep_audit.
17. Zeno records each audited account in SQLite so the same account is not repeatedly audited in later runs.
18. Zeno writes a summary report for the run.
19. Zeno pushes the collected data folder to the configured target repository.
20. Because DEEP_AUDIT_STOP_AFTER_COMPLETE=1, Zeno stops after completing and syncing the audit.

DATA FILES ZENO CREATES
data/deep_audit/accounts.jsonl
One JSON record per audited account. Includes handle, topic, source URL, profile summary, post count, comment count, and run id.

data/deep_audit/posts.jsonl
One JSON record per collected post. Includes handle, post URL, text, metrics, media fields, topic, and run id.

data/deep_audit/comments.jsonl
One JSON record per collected reply/comment. Includes parent post URL, handle, reply data, and run id.

data/deep_audit/summary.md
Human-readable summary of accounts audited, posts collected, comments collected, mission topic, and target repo.

data/zeno_memory.db
SQLite memory database. Stores beliefs, working memory, source registry, and audited account records.

ACCOUNT DEDUPE LOGIC
Zeno has an audited_accounts table in memory.
Before auditing or engaging with an account, it normalizes the account key from handle or source URL.
If the account key already exists, Zeno skips it.
This prevents repeated audits of the same account across rebirths and future runs, as long as the memory database is preserved and synced.

CONFIG CONTROL
To change Zeno's mission, edit zeno_config.txt.

TOPIC controls what Zeno researches.
TARGET_REPO controls where data is saved.

DEEP_AUDIT_POSTS_PER_ACCOUNT controls post collection depth.
DEEP_AUDIT_COMMENTS_PER_POST controls reply/comment collection depth.
DEEP_AUDIT_STOP_AFTER_COMPLETE controls whether Zeno stops after syncing collected data.
